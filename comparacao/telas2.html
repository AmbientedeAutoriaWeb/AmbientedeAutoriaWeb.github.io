Interfaces Comparação das Versões

Criar Projeto

Com a funcionalidade Criar Projeto, o professor pode elaborar os seus projetos de estudos para serem trabalhados com os alunos. Esta funcionalidade é disponível somente para o perfil professor. A seguir são apresentadas as comparações das telas desta funcionalidade.

1ª Versão

Para criar um projeto no ambiente de autoria Web é necessário acessar o menu horizontal do ambiente, apresentado na Figura 1. Este menu é composto pelos itens: Criar Projeto, Listar Projeto, Nova Senha, Ajuda e Sair. Para criar um projeto o professor deve pressionar o item de menu Criar Projeto.


A Figura 2. mostra o formulário com os dados a serem preenchidos para a criação de um projeto de estudo. Este formulário é composto pelos campos Nome, Descrição, Área do Conhecimento e Escola. Os campos do formulário marcados com asterisco '*' são de preenchimento obrigatório. Para que o professor saiba onde clicou no menu horizontal o item 'Projeto' esta destacado. Nesta tela o professor, também, pode interagir com o menu horizontal e com a barra de pesquisa de projetos.


2ª e 3ª Versão

A partir da segunda versão das telas, o menu horizontal foi alterado. Nesta versão o menu horizontal é composto pelos itens Meus Projetos, Criar Projeto e Projetos Publicados, como pode ser visto na Figura 3. Todos os itens de menu e links, nesta versão, estão sublinhados para se destacarem do restante dos textos. Nesta figura também é apresentado o formulário de criação do projeto.


1ª Versão

Ao criar um projeto o professor deve associar conteúdos a ele. Para isso, o professor deve selecionar uma área do conhecimento. Como na Figura 4, a área do conhecimento 'História' esta selecionada e o ambiente apresentou a lista de conteúdos relacionados a esta área na própria tela.


2ª e 3ª Versão

Na segunda versão do ambiente a lista de conteúdos para serem associados ao projeto são apresentados em uma caixa de diálogo para que o professor possa ver quais foram os conteúdos marcados por ele, como na Figura 5.


A partir da segunda versão o ambiente apresenta uma mensagem de confirmação da ação de incluir conteúdos ao projeto, como na Figura 6.


1ª Versão

Para que o professor possa aplicar o projeto criado nas turmas que deseja é necessário associar turmas ao projeto. Para realizar esta ação o professor deve selecionar uma escola para que o ambiente apresente uma lista com as turmas relacionadas a esta escola, como na Figura 7.


2ª e 3ª Versão

A Figura 8. exibie a lista de turmas a serem associadas ao projeto, contudo a partir da segunda versão o ambiente apresenta esta lista em uma caixa de diálogo.


A partir da segunda versão o ambiente apresenta uma mensagem de confirmação para ação de associar turmas ao projeto, como na Figura 9.


3ª Versão

A partir da terceira versão, caso o usuário tente sair da página sem salvar o trabalho em execução, o ambiente apresenta uma mensagem de alerta como na Figura 10.

Excluir Projeto

O ambiente permite que o usuário exclua os projetos associados a ele. A funcionalidade responsável por isso é a Excluir Projeto. Esta funcionalidade é disponível somente para o perfil professor. A seguir são mostradas as comparações telas desta funcionalidade.

1ª Versão

Caso o professor deseja excluir um projeto, ele deve ir até a tela com a lista de projetos relacionados a ele e pressionar o botão X. O ambiente exibirá uma mensagem de confirmação da exclusão do projeto para evitar que o professor cometa uma ação indesejada, como na Figura 32.


2ª e 3ª Versão

A Figura 33. exibe a mensagem de confirmação de exclusão de projeto. Esta mensagem foi modificada na versão dois, para que fosse informado ao professor que ao excluir um projeto não é possível recupera-lo.


Compartilhar Autoria

A funcionalidade Compartilhar Projeto permite que o professor compartilhe seus projetos com outros professores. Ao compartilhar um projeto, o professor possibilita que outros professores editem o seu projeto, tornando assim a autoria do projeto compartilhada. A seguir são apresentadas as comparações das interfaces referentes a esta funcionalidade.

1ª Versão

A Figura 34. apresenta a tela de compartilhamento do projeto. O item do menu Compartilhar encontra-se destacado dos demais para indicar ao usuário onde ele está. Para buscar o professor com quem deseja-se compartilhar o projeto, basta selecionar a escola onde este professor leciona no campo Escola.


2ª Versão

Na segunda versão, o item de menu Compartilhar foi renomiado para Compartilhar Autoria, como na Figura 35, seguindo as orientações dos especialistas durante a avaliação heurística.


3ª Versão

Durante o teste de usabilidade percebeu-se que alguns professores tiveram dificuldade de encontrar o item de menu Compartilhar Autoria, assim o menu horizontal foi alterado para que ficasse mais evidente, como na Figura 36.


1ª Versão

Ao selecionar uma escola o ambiente apresenta uma lista de professores relacionados a ela, como na Figura 37. Para compartilhar o projeto basta o professor marcar os professores com quem deseja compartilhar e pressionar o botão Compartilhar Projeto.


2ª e 3ª Versão

A partir da segunda versão a lista de professores com quem o professor autenticado no ambiente pode compartilhar o projeto aparece dentro de uma caixa de diálogo, como na Figura 38.


A Figura 39. apresenta a mensagem de confirmação do compartilhamento do projeto com os professores selecionados que foi inserida na segunda versão.

Para publicar um projeto o professor deve pressionar o item de menu Publicar. O ambiente irá publicar o projeto e exibir uma mensagem de alerta como na Figura 40.

2ª Versão

A Figura 44. apresenta a mensagem de alerta que orienta o professor a avaliar o projeto antes de publica-lo na segunda versão.

3ª Versão

A Figura 45. apresenta a mensagem de alerta que orienta o professor a avaliar o projeto antes de publica-lo na terceira versão. 1ª Versão

Para pesquisar um projeto cadastrado no ambiente o professor precisa utilizar a barra de pesquisa, como na Figura 46.


2ª e 3ªVersão

Utilizar Projeto

A funcionalidade Utilizar Projeto permite que o professor aplique em suas turmas os projetos publicados por outros professores. A seguir são apresentadas as comparações das interfaces referentes a esta funcionalidade.

1ª Versão

Ao realizar uma pesquisa o professor pode selecionar um projeto para trabalhar com suas turmas. Para utilizar um projeto publicado por outro professor, basta associar turmas a este projeto. A Figura 48. mostra a tela com os dados do projeto e uma área para o professor associar as turmas que deseja aplicar o projeto.


2ª Versão

A Figura 49. apresenta a tela Dados do projeto para a segunda versão.


3ª Versão

A Figura 50. apresenta a tela Dados do projeto para a terceira versão. Como pode ser visto nesta figura, os campos Nome, Descrição e Avaliação são apresentados em textos pois durante o teste de usabilidade os participantes tiveram problemas com os campos desabilitados.


1ª Versão

Ao selecionar uma escola, o ambiente apresenta uma lista contendo todas as turmas da escola que o professor autenticado no ambiente leciona, como na Figura 51.


2ª e 3ª Versão
